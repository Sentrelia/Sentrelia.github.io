Sentrelia Privacy Policy

Last updated: May 30, 2026

This policy explains what Sentrelia stores, what it only uses temporarily, and how to ask for data to be removed.

Sentrelia is a Discord bot for moderation, utility, and fun commands. It uses slash commands and Discord interaction data.

Short version

  • Sentrelia does not read or monitor normal chat messages.
  • Sentrelia does not store message content from normal chat.
  • Sentrelia stores moderation warning/case records when staff use warning commands.
  • Sentrelia does not sell user data.
  • Sentrelia does not use user data to train AI models.
  • QR codes are generated locally by the bot, not by a third-party QR API.

Data Sentrelia may receive

When someone uses a command, Discord sends Sentrelia the information needed to run that command. Depending on the command, this can include:

  • user IDs;
  • server IDs;
  • channel IDs;
  • role IDs;
  • message IDs for moderation actions such as purges;
  • command names and command options;
  • display names, avatars, server names, channel names, role names, and emoji names;
  • Discord permission and role information needed for checks.

This is normal Discord interaction data. Sentrelia uses it to answer the command or perform the requested moderation action.

Data Sentrelia stores

Sentrelia stores moderation warning/case records. These records are created when server staff use commands such as /warn.

A warning/case record can include:

  • case ID;
  • server ID;
  • warned user ID;
  • moderator user ID;
  • warning reason written by the moderator;
  • creation time;
  • whether the case is active or cleared;
  • who cleared it and when, if it was cleared.

This data is stored so server staff can use commands like /warnings, /case, and /clearwarnings.

Data Sentrelia does not store

Sentrelia does not intentionally store:

  • QR code text after the command finishes;
  • calculation expressions after the command finishes;
  • fun command inputs after the command finishes;
  • ordinary chat messages;
  • direct messages;
  • passwords, tokens, payment information, or private health information.

Discord itself may keep command responses or messages according to Discord's own systems and policies.

How data is used

Sentrelia uses data to:

  • run slash commands;
  • check whether a user or the bot has permission to do something;
  • carry out moderation actions requested by server staff;
  • keep moderation warning history;
  • generate QR codes and calculation results;
  • show server, user, role, channel, emoji, avatar, and permission information;
  • send fun command responses.

Sentrelia does not sell personal data and does not use advertising trackers.

Third-party services

Sentrelia uses Discord's API to receive commands and send responses.

The /qr command generates QR images locally in the bot process. Sentrelia does not send QR text to a third-party QR code generator.

The bot may run on hosting infrastructure chosen by the bot operator. Hosting providers may process technical logs needed to run and protect the service.

Retention

Temporary command data is used only long enough to respond to the command.

Moderation warning/case records are kept until one of these happens:

  • server staff clear the warning with a bot command;
  • a server owner or authorized moderator asks for deletion;
  • the bot operator removes old data during maintenance;
  • the bot stops operating and the data is no longer needed.

Some moderation records may be kept when needed for abuse prevention, safety, dispute handling, or legal reasons.

Deletion requests

To request deletion of data connected to you, contact the bot operator through the support server or contact method listed on Sentrelia's Discord application profile.

Helpful details to include:

  • your Discord user ID;
  • the server ID where the data exists;
  • what you want deleted.

For server moderation records, the bot operator may also need to confirm the request with the server owner or authorized staff.

Security

Sentrelia is designed to keep stored data limited. Access to stored bot files should only be available to the bot operator or trusted maintainers.

No online service can promise perfect security. Do not put passwords, tokens, payment information, private health information, or other sensitive information into bot commands.

Age

Sentrelia is intended for people who are allowed to use Discord under Discord's rules and local law.

Changes

This policy may change when Sentrelia changes. The date at the top will be updated when the policy is changed.

Contact

For privacy questions or deletion requests, contact the bot operator through the support server or contact method listed on Sentrelia's Discord application profile.
