Sentrelia Terms of Service

Last updated: May 30, 2026

Sentrelia is a Discord bot for server moderation, utility commands, and small fun commands. These terms are here so server owners, moderators, and users know what is expected when they use the bot.

By adding Sentrelia to a server, installing it as a user app, or using one of its commands, you agree to these terms. If you do not agree, do not use the bot.

Discord comes first

Sentrelia runs on Discord, so Discord's Terms of Service, Community Guidelines, Developer Terms, and server rules still apply. These terms do not replace Discord's rules.

Do not use Sentrelia to harass people, spam servers, evade moderation, collect sensitive information, or break Discord's rules or the law.

What the bot is for

Sentrelia provides commands for things like:

  • moderation actions, such as warnings, timeouts, bans, purges, role tools, channel locks, and channel resets;
  • utility commands, such as user info, server info, role info, timestamps, QR codes, calculations, and permission checks;
  • lightweight fun commands, such as dice rolls, prompts, tic tac toe, and would-you-rather questions.

Sentrelia is not an emergency tool and does not give legal, financial, medical, or professional advice.

Server staff are responsible for setup

Server owners and administrators are responsible for deciding whether Sentrelia is right for their server.

Moderation commands should only be available to trusted staff. Sentrelia uses Discord permissions and role hierarchy checks, but server staff are still responsible for their own role setup and moderation decisions.

If a command fails because of missing permissions or role hierarchy, that usually means the bot's role or the user's role is not high enough, or the bot is missing a Discord permission in that channel or server.

User app commands

Some utility and fun commands can be used when Sentrelia is installed as a user app. Server moderation commands are meant for servers and require the matching Discord permissions.

User-provided content

The /qr command generates QR codes locally from text provided by the user. Users are responsible for the text or URL they choose to encode.

You can stop using Sentrelia by removing it from your server, removing its permissions, or uninstalling it as a user app.
